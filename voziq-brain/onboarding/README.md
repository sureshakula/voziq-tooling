# Onboarding

What a new VOZIQ engineer reads in their first week. This folder is deliberately short; it should link into `decisions/`, `runbooks/`, and `domain/` rather than duplicate them.

Seed this with:

- `week-one.md`: environment setup, accounts, the repos and what each one is
- `who-owns-what.md`: systems mapped to the people who know them best
- `reading-list.md`: the five decisions and five domain notes that explain the most

The best time to improve this folder is during someone's actual onboarding. Give every new engineer a standing task: PR the fixes for everything that confused you.
