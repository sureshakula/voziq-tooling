# Sessions

Distilled learnings from individual work sessions, mostly written by coding agents via `/brain-memo` and reviewed by their engineer before the PR goes up.

These are raw material, not finished knowledge. The bar for merging is lower than elsewhere in the repo: true, clean of secrets and PII, and worth more than nothing in three months. Findability matters less because these notes are meant to be promoted, not referenced forever.

Promotion is the point. When the same lesson appears in two or three session notes, someone turns it into a decision, runbook, or domain note and marks the session notes `status: superseded` with a forward link. A quarterly sweep of this folder is a good habit; a `sessions/` folder that only ever grows is a landfill.

Naming: `YYYY-MM-DD-short-slug.md`.
