# Domain

Facts about VOZIQ's systems and clients that aren't written down anywhere else: model quirks, pipeline behavior, client-specific arrangements, data oddities. The stuff that currently lives in one person's head and surfaces only when they're on vacation.

The test for whether something belongs here: if a teammate learned it the hard way tomorrow, would they be annoyed it wasn't written down?

Use client short codes, never end-customer names or identifiers. If a fact can't be written without customer PII, it doesn't go in this repo.

Copy `TEMPLATE.md`. Keep each note to one fact or one tightly related cluster; ten small findable notes beat one encyclopedia page.
